  • Unit Conversion

    The oscilloscope returns the velocity of the membrane from the LDV in Volt units. We need to translate those units to the velocity or displacement of the membrane.

    To do that we have the following to functions:

    def vamp2disp(vel, freq):
    global vel_range
    return vel*1000*vel_range / (4*3.14*freq)

def damp2disp(disp):
    return disp/2*disp_range
  • Main Loop

    The PySimpleGUI program operates on a window that has to be initialized (we've done that), and a 'main loop' that handles all inputs (e.g. input boxes, radios, combo boxes) and events (e.g. button presses, slider movements, selections). Here is the loop:

    while True:
    event, values = window.read(timeout=200)
    if event == sg.WIN_CLOSED:
        break
    elif event == "Range":
        collect_data(values)
    elif event == "Next":
        current_freq = amplifier.query_ascii_values("FREQ?")[0]
        amplifier.write("FREQ %d" % (current_freq + int(values["DeltaFreq"])))
    elif event == "Init":
        amplifier.write("FREQ %d" % int(values["StartFreq"]))
    elif event == "amplifier_port_selected":
        amplifier = rm.open_resource(values["amplifier_port_selected"])
        sg.Popup("*IDN?", str(amplifier.query("*IDN?")))
    elif event == "oscil_port_selected":
        oscil = rm.open_resource(values["oscil_port_selected"])
        sg.Popup("*IDN?", str(oscil.query("*IDN?")))
    elif event in ["LDV_mode_vel", "LDV_mode_disp"]:
        LDV_mode = event.split("_")[-1]
    elif event == "Open":
        path = values["Path"]
        path = os.path.realpath(path)
        os.startfile(path)

    The events are handled here, for example, when selecting an amplifier port 'amplifier_port_selected' event will trigger and get the amplifier instrument using pyvisa.

  • Range Data Collection

    When pressing the RangeRangeRange﻿ button, the collect_data() function is ran.

    First we get the values passed in from the window, and initialize the lists to store the data:

    vel_range = int(values["VelRange"]) # Initialize data storage
freq_list = []
amp_list = []
disp_list = []
ac1_list = []
ac2_list = []
try: # Get window values
    sf = int(values["StartFreq"])
    ef = int(values["EndFreq"])
    df = int(values["DeltaFreq"])
    t = float(values["TimePerFreq"])
except:
    sg.Popup("Error", "Invalid Data Types: Need to be {Int, Int, Int, Float}")
    return

    Next we need to loop over the Frequency values and send them to the lock-in-amplifier. Then we get the Amplitude values back from the oscilloscope in voltage, and translate them using the unit conversion functions.

    for freq in range(sf, ef+1, df):
		amplifier.write("FREQ %d" % freq) # Set amplifier frequency
		freq_list.append(freq) # add current frequency
		oscil.write(':SINGle') # Update oscilloscope
		amp_list.append(oscil.query_ascii_values(':MEASure:ITEM? Vpp,CHANnel'+values["ampChannel"])[0]) # Read oscilloscope vel amplitude
		if values["saveAC"]: # Save AC current from oscilloscope
		    ac1_list.append(oscil.query_ascii_values(':MEASure:ITEM? Vpp,CHANnel'+values["ac1Channel"])[0])
		    ac2_list.append(oscil.query_ascii_values(':MEASure:ITEM? Vpp,CHANnel'+values["ac2Channel"])[0])
		disp = vamp2disp(amp_list[-1],freq) if LDV_mode=="vel" else damp2disp(amp_list[-1]) # translate to correct units
		disp_list.append(disp) # add final value to list
		sleep(t) # (delay between measurements)

    To do that, we use the pyvisa library.

    Then we filter the values to remove ones which don't fit (too big/small, those occur when the oscilloscope range doesn't match the values). We also calculate some values such as ImaxI_{max}Imax​﻿, frf_rfr​﻿, and QQQ﻿ (I won't go over the calculation.)

Operating the UI

Measuring

  • Start Frequency, End Frequency, and Delta set the range of frequencies to be sent to the amplifier. For example 400, 1000, 25 will result in [400,425,450...950,975,1000][400, 425, 450...950, 975, 1000][400,425,450...950,975,1000]﻿.
  • Delay is the delay in seconds between each two frequency measurements.
  • Vel Range is the LDV velocity range (displacement range in case of Displacement mode). Input it matching the LDV mode set in the LDV settings.
  • Range Button will run the function collect_data() which will input the frequency range to the amplifier, collects the oscilloscope amplitude values back and saves it all to a file.
  • Init Button will set the amplifier frequency to the Start Frequency frequency.
  • Next Button will add Delta to the current amplifier frequency.

File Settings

  • File Path determines where the saved file from the Range function will output. Cannot be a shared directory (a server). Press Browse to set the path.
  • File Name will determine the file's name (filename.txt). $m will be replaced the the LDV mode (vel or disp) and $t will be replaced with the time. Beware not to use the characters \ /:*?"<>| because those cannot be in a file name (will result in an error). Press Open to open the text file location.

Settings

  • Amplifier Port, and Oscilloscope Port are a dropdown menu from which you need to select the port for the instruments. When selecting, the command *IDN? will be sent to the selected instrument. A popup for the result of the query will show with its reply from the instrument - that way you can know if you selected correctly. Sometimes an incorrect selection can result in an error if the instrument does not have an *IDN? command.
  • Vel/Disp will select the LDV measurement mode.
  • SaveAC selects whether to save AC values while a measurement is happening.

Channels

Input only values of 1-4. These are the channels from which the program will collect the data from the oscilloscope.
